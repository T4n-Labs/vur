# VUR (Void User Repository) Documentation

This documentation explains **how to use VUR** and **how to create package templates** for VUR using `xbps-src`. It is intended for Void Linux users, developers, and package maintainers.

## Prerequisites

Before using VUR, ensure that:

  * You are using **Void Linux**
  * `git` is installed
  * `xbps-src` is installed

<!-- end list -->

```sh
sudo xbps-install -S git xtools
```

Ensure `xbps-src` is configured:

```sh
xbps-src binary-bootstrap
```

-----

## How to Use VUR

### 1\. Clone the VUR Repository

```sh
git clone https://gitlab.com/vur/vur.git
cd vur
```

Main structure:

```
VUR/
├── core/
├── extra/
└── multilib/
```

-----

### 2\. Build a Package from VUR

For example, to build the `foo` package from the `extra` category:

```sh
cd extra/foo
xbps-src pkg foo
```

The build results will be located in:

```
hostdir/binpkgs/
```

-----

### 3\. Install a Package from VUR

```sh
sudo xbps-install --repository=hostdir/binpkgs foo
```

Alternatively, to install with all dependencies automatically:

```sh
sudo xbps-install --repository=hostdir/binpkgs foo
```

## How to Create a VUR Package Template

### 1\. Determine the Category

Choose the template location:

  * `core/` → System / essential packages
  * `extra/` → Additional software
  * `multilib/` → 32-bit packages

Example:

```sh
cd extra
mkdir hello-vur
cd hello-vur
```

-----

### 2\. Create the `template` File

A `template` file is **mandatory**.

Minimal structure:

```sh
pkgname=hello-vur
version=1.0.0
revision=1
build_style=gnu-makefile
short_desc="VUR package example"
maintainer="Your Name <you@email.com>"
license="MIT"
homepage="https://example.com"
distfiles="https://example.com/hello-${version}.tar.gz"
checksum=SKIP
```

-----

### 3\. Explanation of Key Fields

  * **pkgname**: Package name (lowercase, unique)
  * **version**: Upstream version
  * **revision**: Template revision (starts at 1)
  * **build\_style**: Build method (`gnu-makefile`, `cmake`, `python3-module`, etc.)
  * **short\_desc**: Brief description
  * **maintainer**: Person responsible for the package
  * **license**: Software license
  * **distfiles**: Upstream source code
  * **checksum**: SHA256 of the source (avoid `SKIP` whenever possible)

-----

### 4\. Build & Test the Template

From the VUR root:

```sh
xbps-src pkg hello-vur
```

If errors occur, fix them until the build succeeds.

-----

### 5\. Adding Supplemental Files (Optional)

If you need patches or additional files:

```
hello-vur/
├── template
└── files/
    └── fix-path.patch
```

Patches will be automatically applied by `xbps-src`.

## Common Build Styles

  * `gnu-makefile`
  * `cmake`
  * `meson`
  * `python3-module`
  * `cargo`
  * `go`

Python example:

```sh
build_style=python3-module
```

-----

## Rules & Best Practices

  * Do not duplicate packages from the official Void repository.
  * Use official upstream sources.
  * Do not include prebuilt binaries.
  * Ensure the template can be built in a clean environment.
  * Update the template if the upstream changes.

## Troubleshooting

### Source Download Failed

  * Check the `distfiles` URL.
  * Ensure the version is correct.

### Incorrect Checksum

```sh
xbps-src fetch
xbps-src verify
```
